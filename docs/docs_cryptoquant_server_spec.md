## 二、三档配置

### 档 A · 最小原型研究（阶段 0–2，零依赖）
> 当前沙盒环境已基本满足，仅需略升配。

- **CPU**：8 vCPU
- **内存**：16–32 GB
- **GPU**：无
- **存储**：100 GB SSD（数据 + 进化种群 + WFA 结果）
- **网络**：普通公网
- **月成本**：约 ¥150–400（轻云实例）
- **适用**：规则/验证/进化/规则对手全跑得动；不含 LLM 与 TSFM

### 档 B · 完整蓝图自托管（阶段 3–4 全含，LLM+TSFM+DRL 都本地）
> "全部完成"的完整自托管形态。

- **CPU**：32–64 vCPU（编排 + GP 进化 + 数据 + 执行）
- **内存**：128–256 GB
- **GPU**：2–4 张
  - 1× A100 80GB（World Model/DRL 训练 + TSFM）
  - 1× L4 / A10G 24GB（LLM 4 角色 + StockSim LLM 市场）
  - 或简化：单台 4×A100 80GB 节点（统一调度）
- **存储**：1–2 TB NVMe（行情 + 模型权重 + 向量 DB + 日志）
- **向量 DB**：Chroma / Qdrant / pgvector（FinMem 检索，占用小）
- **网络**：1–10 Gbps，低延迟到交易所；内网 GPU 节点高速互联
- **月成本**：约 ¥1.5万–4万（云 GPU 实例，按 4×A100 估；L4 档可压到 ¥8千–1.5万）
- **适用**：完整自托管、数据不出域、最可控

### 档 C · 完整蓝图（LLM 走 API，其余自托管）
> 折中：LLM 用 Claude/GPT API，省掉 LLM GPU。

- **CPU / 内存 / 存储**：同档 B（略降）
- **GPU**：1× A100 80GB 或 1× L4 24GB（仅 TSFM + DRL 训练/推理）
- **额外**：LLM API 调用费（按调用量，约 ¥3千–3万/月，取决于信号频率）
- **月成本**：GPU ¥8千–1.5万 + API ¥3千–3万
- **适用**：不想养 LLM 显卡、接受数据出域

## 三、实盘执行层（独立于研究，翻 Gate A–F 后才需要）

> 蓝图完成 ≠ 实盘。实盘是**独立硬化层**：算力轻、可靠性与合规重。

- **算力**：2–4 vCPU / 8–16 GB（执行引擎本身不重）
- **关键不是算力，是**：
  - 多可用区双机热备 + KillSwitch 独立进程
  - 交易所专线 / VPN，低延迟网络
  - 密钥 HSM / Secret Manager（非明文 key，如我们刚用的 ED25519）
  - 审计日志 + 合规（EU AI Act 等可解释要求）
  - 人工 ACK 闸门（manual_resume）
- **月成本**：¥500–2000（轻实例 + 冗余），远低于研究层
- **结论**：实盘贵在可靠性与合规，不贵在算力

## 六、基于你现有资产的推荐

你已有：腾讯云 SCF(函数计算) + COS(`contract-signals-1449535414`) + CLS + SecretId/Key。

- **研究训练**：用腾讯云 **GPU 实例**（如 GN 系列 A100/L20）跑 TSFM/DRL，结果存 COS，日志进 CLS。
- **周期 WFA**：复用 SCF 定时触发（你合约系统已是每 4h 任务），跑 `run_wfa_v2.py` 改造版。
- **常驻多智能体**：用容器服务（TKE）跑 4 角色 + metacontroller，不全天占 GPU——LLM 走 API、TSFM 推理按需起实例。
- **起步建议**：先从**档 A** 把阶段0–2 跑通（当前沙盒加一点内存即可），阶段3–4 再按需开 GPU 实例，避免一上来养 4×A100。

> 核心原则：蓝图全量完成 = 研究全栈，算力集中在 GPU（TSFM/DRL/LLM）；实盘是另一回事，轻算力 + 高可靠。不要为研究层的 GPU 账单吓到——阶段0–2 根本不用 GPU。
