## 五、跨阶段铁律（每阶段进门强制）

1. **验证层不可逾越**：每阶段进门强制过 Purged+Embargo CV + DSR/PSR + SPCI 覆盖 + 受控 A/B 显著（来源：实施计划 §七）。
2. **依赖纪律**：阶段0–2 零新依赖（numpy/stdlib + 已有数据即可跑）；torch/LLM 后移至阶段3–4，且必须带降级路径（torch 缺失回退 numpy）。
3. **实施顺序**：阶段0 → 0.5 → 1 → 2 → 3 → 4。验证基建（0.5）必须先于或等于一切 ML 进门。
4. **不碰实盘**：全程原型沙盒，真钱由宪法 R0 + Go/No-Go Gate A–F 锁死。
